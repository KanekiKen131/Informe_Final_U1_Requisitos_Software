# Proyecto Final U1 – Requisitos de Software

## Descripción del Proyecto
Este proyecto corresponde al trabajo final de la Unidad 1 de la asignatura **Requisitos de Software**.  
El objetivo principal es analizar, modelar y documentar los requerimientos de un sistema para las practicas preprofesionales  mediante diferentes diagramas y prototipos que permitan representar el funcionamiento y las necesidades del software.

---

## Objetivos

### Objetivo General
Desarrollar el análisis de requisitos de un sistema de software utilizando herramientas de modelado y documentación.

### Objetivos Específicos
- Identificar los requerimientos funcionales y no funcionales.
- Elaborar diagramas de casos de uso.
- Diseñar prototipos de interfaz.
- Aplicar técnicas de modelado de software.
- Documentar adecuadamente el proyecto.

---

## Contenido del Proyecto
El proyecto incluye:

- Diagramas de casos de uso.
- Especificación de requisitos.
- Prototipos de interfaces.
- Análisis del sistema.
- Documentación técnica.
